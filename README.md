# dl-2026-05

## 题目名称

钢铁缺陷检测科研方案写作：完成技术路线与答辩材料

## 这是一道什么题

这道题主要考查：

- 科研 / 竞赛材料结构化写作能力
- Word 排版与图表规范
- 技术方案表达能力
- 文献阅读与引用规范
- 逻辑自洽、风险意识、工作量规划
- 压力性追问回答、方案边界说明与答辩抗压能力

## 题目背景

请你围绕第三、四题完成一份 Word 技术方案书：

**基于深度学习的钢铁表面缺陷检测方法的研究与分析**

方案必须与前面题目中的钢铁缺陷检测任务呼应，重点围绕 NEU-DET、GC10-DET、Severstal 钢铁缺陷数据集、YOLO 系列检测模型、数据清洗增强、检测指标、竞赛提交与工程落地展开。

本题不接受泛泛而谈的“智能化赋能工业质检”文章。所有结论必须能被数据集、论文、实验设计或工程约束支撑。

由于该方向的几位评审均有较丰富的工业检测模型研究和设计经验，因此你的回答应当较为具体。压力性答辩问题不得直接使用 AI 生成答案，否则该部分按 0 分处理。

文档最终提交：

- `proposal.docx`
- `proposal.pdf`
- `references.bib` 或 `references.md`
- `defense_questions.md`
- `figures/` 原始图片或图表文件
- `layout_checklist.md`

## 你的任务

`proposal.docx` 至少包含以下一级标题，顺序不得随意打乱：

- 研究背景与意义

- 实验设计

- 正文行距固定为 1.5 倍或 22 磅，全文必须统一
- 一级标题黑体三号或小三号，二级标题黑体四号，三级标题黑体小四号
- 段前段后间距统一，正文首行缩进 2 字符

- 交叉引用或正文引用必须能对应到图表编号
- 同时导出 PDF，PDF 不得出现目录页码错乱、图片缺失、公式乱码

评分：

- 结构完整且顺序正确，4 分
- 自动目录、标题样式、页码规范，3 分
- 字体、字号、行距、缩进、页边距统一，3 分
- 图表编号、题注、交叉引用规范，3 分

### 2. 技术背景与文献综述，10 分

要求：

- 每类方法必须写出：代表方法、输入输出、优点、局限、适用场景、为什么适合或不适合钢铁表面缺陷
- 不得只罗列论文标题，不得只写“提高准确率、增强鲁棒性”等空话

### 3. 数据集、指标与实验方案设计，20 分

要求：

- 明确 NEU-DET、GC10 或 Severstal 中至少 2 个数据集的来源、类别、规模、图像尺寸、标注形式、已知缺陷，需要至少一个三线表，若使用Origin画图更好，使用Origin图表示清晰正确者额外加3分

- 设计至少 5 组对比实验，数据可以来自权威参考文献所报告的数值
- 设计至少 5 组消融实验，例如输入尺寸、增强策略、小目标过滤阈值、是否使用 Mosaic、不同 YOLO 结构、不同损失函数、不同数据集混训策略

- 划分方案和风险讨论合理，3 分

- baseline、对比实验、消融实验设计完整，5 分

### 5. 压力性答辩问题逐题回答，30 分

请提交 `defense_questions.md`，并在 Word 附录中加入“压力性答辩问题回答”。

直接回答下面给出的追问。每一题都必须结合自己的方案书回答，不能泛泛而谈，不能直接使用 AI 生成答案。回答必须包含至少 1 个明确依据，例如正文页码、图表编号、参考文献编号、数据集统计、实验设计、工程约束或可执行验证步骤。

回答格式：

- 每题 150-300 字。
- 每题必须先给出正面回答，再说明依据，最后说明如果判断错误如何验证或补救。
- 不得只写“后续优化”“进一步研究”“提高鲁棒性”等空话。
- 如果某题无法回答，必须承认方案书中缺少什么，并说明补救实验或补充材料。
- 不得使用 AI 直接回答；如果回答明显模板化、无法对应自己的正文依据，该题不得分。

请逐题回答以下 24 个压力性问题：

1. 你的方案真正要解决的核心问题是什么？它和“拿一个 YOLO 跑钢铁缺陷数据集”有什么本质区别？
2. 你的创新点到底是什么？哪些是方法创新，哪些只是工程流程整理或参数调整？
3. 如果评委认为你的创新点只是普通调参，你如何反驳？请指出方案中最有说服力的一处证据。
4. 你为什么选择 YOLO？为什么不是 Faster R-CNN、Mask R-CNN、U-Net、SegFormer 或传统机器视觉？
5. 如果缺陷边界不规则，检测框是否会损失关键信息？什么情况下你必须改用分割模型？
6. 你的数据集选择是否足以代表真实钢厂场景？NEU-DET、GC10-DET、Severstal 与现场数据最大的差异可能是什么？
7. 如果公开数据集和现场钢板纹理、光照、相机角度差异很大，你的方案如何迁移？
8. 如果数据集中存在重复图像或同一钢板相邻裁剪同时进入 train 和 test，你的准确率还有意义吗？你如何排查？
9. 如果类别极度不均衡，mAP 看起来不错但少数类几乎检测不到，你会如何发现并修正？
10. 如果标注框存在系统性偏移、漏标或误标，你如何判断问题来自标注而不是模型？
11. 你设计的 train / val / test 划分为什么合理？如果重新划分后指标大幅下降，说明什么？
12. 从零训练 100 轮效果很差时，你的方案是否还成立？你如何区分是训练不足、数据不足、模型不合适还是标注有问题？
13. 如果预训练模型明显优于从零训练，你为什么还要设计从零训练 baseline？
14. 你的评价指标为什么选这些？mAP@0.5、mAP@0.5:0.95、Recall、FPS 分别回答什么问题？
15. 如果 mAP 很高但现场误检严重，你如何定位是数据问题、指标问题、模型问题还是部署预处理问题？
16. 如果 Recall 提高但 Precision 明显下降，工业质检场景下你会怎么取舍？为什么？
17. 如果某个消融实验只提升 0.3 mAP，你如何判断它是真提升还是随机波动？
18. 你的对比实验是否公平？不同模型输入尺寸、训练轮数、增强策略、预训练权重不一致时怎么处理？
19. 你引用的论文真的支持你的结论吗？请举一个最关键结论，并说明对应论文支持到什么程度。
20. 你的技术路线图中哪个环节最容易失败？失败后第一个排查动作是什么？
21. 如果只剩 7 天参赛，你会砍掉哪些实验，保留哪些实验？你的取舍标准是什么？
22. 如果部署设备算力不足，你优先牺牲输入尺寸、模型规模、精度、召回率还是推理速度？为什么？
23. 如果评委要求你明天拿出一个最小可运行系统，你能交付什么？不能交付什么？
24. 你方案中最薄弱的一处是什么？如果让你重写方案，你会优先补强哪一部分？

- 24 个问题全部作答且能对应自己的方案，24分
- 每题有明确依据，不脱离正文、图表、文献或数据集，3分
